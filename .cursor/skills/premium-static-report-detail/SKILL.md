# 🌥 Cloud Between Us: Premium Toolkit Architecture (Final)

## 🎯 문서 개요

본 문서는 MVP 단계에서 $2.99 매출 검증을 위한 유료 콘텐츠의 구조와 데이터를 정의한다.
단순 감성 묘사를 넘어, 유저가 **"우리의 갈등 원인과 해결책을 찾았다"**고 느끼게 만드는 **[실전 대응 매뉴얼]**에 초점을 맞춘다.

## 1️⃣ 6개 타입별 코어 데이터 (The Engine)

_모든 조합은 아래 6개 타입의 데이터를 조합 규칙에 따라 교차 생성한다._

| 타입        | 명칭 | Pace    | 핵심 추진력 (Drive)   | 근원적 두려움 (Fear)    | 갈등 반응 (Reaction)     | 회복을 위한 필요 (Needs)  |
| :---------- | :--- | :------ | :-------------------- | :---------------------- | :----------------------- | :------------------------ |
| **Sunlit**  | 햇살 | Fast    | 명확함, 따뜻한 방향성 | 무시당함, 정체됨        | 빨리 해결하려고 몰아붙임 | 빠른 확인과 확신          |
| **Mist**    | 안개 | Slow    | 깊이, 감정의 층위     | 압박, 오해받음          | 침묵하며 내부로 침잠     | 충분한 시간과 부드러운 톤 |
| **Storm**   | 폭풍 | Intense | 솔직함, 에너지 분출   | 가식, 연결의 단절       | 감정을 날것으로 터뜨림   | 격렬한 감정의 수용과 인정 |
| **Cloud**   | 구름 | Steady  | 안정감, 평온한 유지   | 갈등 그 자체, 평화 파괴 | 회피하거나 감정을 억제   | 안전하다는 물리적 신호    |
| **Wind**    | 바람 | Fluid   | 자유, 새로운 영감     | 구속, 감정적 무게감     | 주제를 돌리거나 가벼워짐 | 거리둠의 존중과 유머      |
| **Shelter** | 숲   | Still   | 수용, 단단한 지지     | 거절, 쓸모없어짐        | 과하게 맞춰주다 지침     | 존재 자체에 대한 감사     |

## 2️⃣ 유료 리포트 구성 (6-Block Structure)

### Block 1. The Core Truth (관계의 본질)

- **로직:** `User.Drive` + `Partner.Drive` + `Pace 대비`
- **내용:** 두 에너지가 만났을 때 형성되는 고유 기후 현상 설명. (예: 안개 사이로 스며드는 아침빛)

### Block 2. Your Conflict Blueprint (갈등 설계도)

- **로직:** `User.Reaction` ↔ `Partner.Reaction` | `User.Fear` ↔ `Partner.Fear`
- **내용:** "당신은 {무시당할까 봐} {몰아붙이고}, 상대는 {압박감에} {입을 닫습니다}."와 같이 갈등의 악순환을 Fear 중심으로 해석.

### Block 3. Real-Life Scenarios (실전 상황 대응법)

- **상황 A (상대가 반응할 때):** 상대의 `Reaction` 시 내가 해야 할 구체적 행동.
- **상황 B (갈등이 반복될 때):** 둘 사이의 흐름을 끊는 질문 제안.
- **Script:** "압박하려는 게 아니야. 우리 사이가 멀어질까 봐 걱정돼서 그래." (상대의 `Needs` 기반)

### Block 4. The Emotional Reset (감정 리셋 프레임워크)

- **1단계 (24h):** 안정화. `Pace` 차이를 인정하고 감정 온도 낮추기.
- **2단계 (48h):** 각자 `Fear` 고백하기. "사실 내가 두려웠던 건..."
- **3단계:** `Needs` 채워주기. 물리적 근접성 또는 인정의 말.

### Block 5. Your Personal Repair Line (수리 문장)

- **For You:** "맞는지보다, 우리가 더 중요해."
- **For Partner:** "나는 떠나지 않아. 잠시 숨 고를 시간이 필요해."

### Block 6. 30-Day Climate Shift (장기 기후 조정)

- **1~4주차:** `Pace` 관찰 → `Needs` 질문법 연습 → 주간 체크인 → 안정감 확인 질문.

## 3️⃣ [Case Study] 핵심 조합별 마찰 해결 로직

### Case A: Fast(햇살) x Slow(안개/구름)

- **문제:** 한 명은 숨이 차고, 한 명은 답답함.
- **해결:** '시간'을 약속할 것. Fast는 기다리고, Slow는 침묵 대신 "생각할 시간이 필요함"을 명시할 것.

### Case B: Intense(폭풍) x Still(숲/구름)

- **문제:** 한 명은 폭발하고, 한 명은 얼어붙음.
- **해결:** '온도'를 분리할 것. Intense는 비난을 멈추고 에너지만 발산하고, Still은 상대의 화를 자신의 탓으로 돌리지 말 것.

### Case C: Fluid(바람) x Steady(구름/숲)

- **문제:** 한 명은 떠나려 하고, 한 명은 붙잡으려 함.
- **해결:** '지점'을 정할 것. Fluid는 돌아올 시간을 약속하고, Steady는 상대에게 날아갈 공간을 허용할 것.
